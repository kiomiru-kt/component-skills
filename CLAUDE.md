# component-skills — AI 行動ルール

## 学びの蓄積と自己改善

このリポジトリはスキル集であり、実際のプロジェクトで得た知見を継続的に反映する。

- セッションで新しい知見（バグ・エッジケース・ベストプラクティス）が得られた場合、**そのセッション内で SKILL.md または scripts/ に反映する**
- 反映内容は具体的に記述する（「rgba対応」「--page Components を必ず指定」等）
- 反映後は必ず PR を作成してマージする（直接 main へのプッシュ禁止）

## PR 必須ルール

- **main への直接プッシュ禁止**
- 変更はすべてフィーチャーブランチで行い、PR を作成してからマージする
- ブランチ命名: `feat/`, `fix/`, `docs/`, `refactor/` プレフィックスを使用
- PR には変更の背景・理由を記述する

## figma-tokenize スクリプト（scripts/figma-tokenize.py）

- `--page Components` は必須（ワイヤー等の他ページを混入させない）
- `--threshold 1` は色に必須（全色が変数管理対象のため出現数が少なくても抽出する）
- rgba カラーはスクリプトが自動抽出（fill opacity × color alpha を合成）
- Figma ノードに fill が設定されていない色は自動取得不可 → `source: "manual"` で手動追記

## 変更時の確認事項

- SKILL.md を変更した場合: `docs/` の関連仕様書も必要に応じて更新する
- scripts/ を変更した場合: `tests/` のテストが通ることを確認してからコミット
